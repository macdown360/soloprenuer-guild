# ソロプレナー・ギルド プロダクトガイドライン

## プロダクトの核

ソロプレナー・ギルドは、一人起業家、フリーランス、個人開発者、副業家が集まるオンライン上の冒険者ギルドです。

案件マッチングサイトではなく、ソロプレナー同士の助け合いと成長を促進するコミュニティプラットフォームとして設計します。

## MVPで最優先する循環

1. クエストを投稿する
2. 他の冒険者が応募し、達成する
3. 発行者が完了を承認する
4. Gold報酬とTrustが反映される
5. 相互評価によって信頼が積み上がる

この「クエスト投稿 → クエスト達成 → 信頼獲得」の循環を最重要指標にします。

## 表現ルール（内部仕様と外部向けの分離）

- 外部向け（ユーザーに見せる画面）では、運営都合の語を出さない
- 外部向けで避ける語: MVP、MAU、KPI、異常検知、連打制限、評価操作対策ロジック
- 外部向けでは、体験中心の語に置き換える
  - 例: MVP目標値 → 今月のギルド活動
  - 例: 異常検知 → 不審な評価は運営確認の対象
- 内部仕様（設計書、運用手順、分析基盤）では、数値指標や検知ロジックを明記する
- 画面文言は「ユーザーの行動」と「得られる価値」を先に書き、実装事情は書かない

## 実装する機能

### 会員登録

- 冒険者名
- メールアドレス
- パスワード
- 任意プロフィール: 職業、スキル、自己紹介、SNS、Webサイト

### マイページ

- 冒険者ランク
- Gold残高
- Trust
- 完了クエスト数
- 発行クエスト数
- プロフィール

### クエスト

- 作成: タイトル、詳細説明、報酬Gold、カテゴリ、締切
- 一覧: タイトル、発行者、報酬、締切、応募人数
- 検索: カテゴリ、キーワード
- 詳細: 内容、発行者情報、応募ボタン、コメント
- 完了: 発行者が承認し、報酬とTrustを付与

### 信頼システム

- クエスト完了で +5 Trust
- 5段階評価で追加Trustを付与
- Trustは譲渡不可、売買不可、換金不可

評価ごとの追加Trustは以下で統一します。

- ★★★★★: +10
- ★★★★: +8
- ★★★: +5
- ★★: +2
- ★: +0

## ポイント設計

### Gold

依頼を出すための通貨です。

- 新規登録時に100G付与
- クエスト発行時に報酬分を確保
- 達成承認後、達成者が報酬を100%受け取る
- ログインボーナスやイベント参加でも獲得可能

### Trust

信用スコアです。

- ランク決定に使う
- 冒険者評価として表示する
- 蓄積型で、通常の依頼発行では消費しない

MVPの最重要ルールは「Goldは増減する、Trustは蓄積する」です。

## ランク

- 見習い冒険者: 0-99
- 駆け出し冒険者: 100-299
- 一人前冒険者: 300-699
- 熟練冒険者: 700-1499
- 上級冒険者: 1500-2999
- マスター冒険者: 3000-4999
- 伝説の冒険者: 5000以上

ランク判定ルールは段階的に運用します。

- MVP: Trustのみでランク判定
- 将来拡張: Trustを主軸に、完了クエスト数と継続利用日数を加点要素として追加

## 運用参照: 実装準拠の算出ルール

この章は、現在の実装と同じルールを運用時に参照するための固定仕様です。
実装ソースは `app.js` を正とし、この章は変更時に必ず同時更新します。

### 1) ランク判定（Trustのみ）

現在の判定関数は `getRank(trust)` で、閾値は以下です。

- 見習い冒険者: 0-99
- 駆け出し冒険者: 100-299
- 一人前冒険者: 300-699
- 熟練冒険者: 700-1499
- 上級冒険者: 1500-2999
- マスター冒険者: 3000-4999
- 伝説の冒険者: 5000以上

### 2) Gold / Trust の加算・減算イベント

現在の `dashboard` 実装で発生する値変動は以下です。

1. クエスト発行（`questForm` 送信）
- 条件: 発行報酬 `reward` が現在Gold以下
- 変動: `Gold -= reward`, `発行クエスト数 += 1`
- 備考: Trustは変動なし

2. クエスト承認（`reviewForm` 送信）
- 変動: `Gold += reward`
- 変動: `Trust += (5 + 評価ボーナス)`
- 評価ボーナス: ★5=+10, ★4=+8, ★3=+5, ★2=+2, ★1=+0
- 変動: `完了クエスト数 += 1`

3. デイリーミッション（各ミッション初回達成時のみ）
- ミッション1: `Trust +5`
- ミッション2: `Trust +8, Gold +3`
- ミッション3: `Trust +3`
- 全3件達成ボーナス: `Trust +5`
- 備考: 同一ミッションの再達成加算はなし（`state.missions[index]` が真なら処理しない）

4. 週次チャレンジ
- 進捗加算トリガー: 承認実行時に `weeklyProgress += 1`（上限3）
- 達成条件: `weeklyProgress >= 3`
- 達成時報酬: `Gold +30, Trust +25`
- 達成後: `weeklyProgress = 0` にリセット

### 3) 称号（現状仕様）

称号表示は現状、`dashboard.html` のカードで静的表示です。
条件文言はUI上に記載済みですが、称号の自動解放ロジックは未実装です。

主な条件文言（UI表示値）

- 初陣: 初クエスト達成
- 助っ人: 10件達成
- 常連冒険者: 7日継続
- 壁打ち名人: 相談回答5件
- 五つ星の協力者: 5つ星評価3件
- 依頼上手: クエスト発行5件

### 4) 運用時チェックポイント

数値事故防止のため、運用前後で以下を確認します。

1. `app.js` の閾値・加算値と本章の数値が一致している
2. 評価プルダウンの表示値と `reviewForm` の加算値が一致している
3. ミッション報酬と週次報酬の重複付与条件が意図通りである
4. 称号条件を動的化する場合は、先に本章へ算出式を追記する

- NFT
- 独自トークン
- AIエージェント

## デザイン方針

- ゲーム感はあるが子供っぽくしない
- 中世ファンタジーの冒険者ギルド感と、SaaSの使いやすさを両立する
- 配色はダークネイビー、ゴールド、シルバーを中心にする
- カード、フォーム、一覧、検索、詳細の操作は業務ツールとして分かりやすくする
