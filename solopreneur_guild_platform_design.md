# ソロプレナーギルド プラットフォーム設計

## 1. 概要

ソロプレナーギルドは、一人起業家、フリーランス、個人開発者、副業家が集まるオンライン上の冒険者ギルドです。

ユーザーは冒険者として登録し、自らクエストを発行したり、他の冒険者のクエストを達成したりすることで、GoldとTrustを獲得します。

本サービスは案件マッチングサイトではありません。小さな依頼と相互評価を通じて、ソロプレナー同士の助け合い、成長、信頼形成を促進するコミュニティプラットフォームです。

## 2. 解決する課題

- 壁打ち相手がいない
- ユーザーフィードバックが集まらない
- 孤独になりやすい
- 仲間不足
- 小さな依頼を気軽に頼めない
- 信頼できる人脈が作りにくい

## 3. MVPの目的

最初の目的は、以下の循環を成立させることです。

1. クエストを投稿できる
2. クエストを達成できる
3. 発行者が完了を承認できる
4. 報酬GoldとTrustが反映される
5. 相互評価で信頼が積み上がる

最重要指標は「クエスト投稿 → クエスト達成 → 信頼獲得」の継続率です。

## 5. ポイント設計

### Gold

依頼を出すための通貨です。

- 新規登録時に100G付与
- クエスト発行時に報酬分を確保
- クエスト達成承認後、達成者へ100%付与
- ログインボーナスは1日1回5G
- 7日連続で+20G、30日連続で+100G

### Trust

譲渡不可、売買不可の信用スコアです。

- クエスト完了で+5
- 5つ星で+10
- 4つ星で+8
- 3つ星で+5
- 2つ星で+2
- 1つ星で+0
- 初回達成で+20
- 月間MVPで+100

Goldは増減し、Trustは蓄積します。この分離を崩さないことが設計上の最重要ルールです。

## 6. ランク制度

- 見習い冒険者: 0-99 Trust
- 駆け出し冒険者: 100-299 Trust
- 一人前冒険者: 300-699 Trust
- 熟練冒険者: 700-1499 Trust
- 上級冒険者: 1500-2999 Trust
- マスター冒険者: 3000-4999 Trust
- 伝説の冒険者: 5000 Trust以上

ランク判定では、Trustに加えて完了クエスト数と継続利用日数も将来的に考慮します。

## 8. 不正対策

- Trustの譲渡禁止
- GoldとTrustの売買禁止
- 自己評価禁止
- 同一ユーザー間の評価連打禁止
- 相互評価の異常検知
- 複数アカウントによる評価操作の制限
- 虚偽の完了報告へのTrust取り消し

## 9. 将来機能

MVPでは以下を実装しません。

- ユーザー作成ギルド
- クラン機能
- パーティ機能
- 通話機能
- 決済機能
- NFT
- 独自トークン
- AIエージェント

まずはクエストと信頼の循環を成立させます。

## 10. 文言レイヤー設計

内部仕様と外部向け文言が混在しないよう、以下を固定ルールとします。

### 外部向け（ユーザー画面・公開ページ）

- 目的: 行動を促し、安心して利用できる状態を作る
- 使用する語: 冒険者、クエスト、Gold、Trust、ランク、承認、評価
- 出さない語: MVP、KPI、MAU、異常検知ロジック、スコア算出式、運営監視パラメータ
- 表現方針: 「何ができるか」「何が得られるか」を先に示す

### 内部向け（設計書・運用資料・分析）

- 目的: 実装と運営判断を再現可能にする
- 明記する内容: 指標定義、算出式、不正検知条件、運用フロー、将来拡張方針
- 注意点: 内部資料で使う語を、画面文言へそのまま流用しない

### 用語マッピング

- MVP目標値 -> 外部では「今月の活動状況」
- MAU -> 外部では「今月アクティブな冒険者数」
- 異常検知 -> 外部では「不審な評価は運営確認の対象」
- 評価連打禁止 -> 外部では「短期間の連続評価には制限があります」
